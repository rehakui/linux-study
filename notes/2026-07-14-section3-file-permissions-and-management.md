# Linuxのディレクトリ構造

Linuxでは各ユーザーのホームディレクトリは `/home` 配下に作成される。

例

```text
/
└── home
    └── haruka
```

インストール直後はホームディレクトリが空の場合もある。

```bash
pwd
# /home/haruka

ls
# （何も表示されない）

ls -l
# total 0
```

`ls` は隠しファイルを表示しないため、

```bash
ls -la
```

で `.bashrc` や `.bash_profile` の有無も確認できる。

# umask

新しく作成されるファイル・ディレクトリのアクセス権は

```
デフォルト権限 - umask
```

で決まる。

### デフォルト権限

|対象|デフォルト|
|---|---|
|ファイル|666|
|ディレクトリ|777|

---

## 現在のumask確認

```bash
umask
```

自分の環境（AlmaLinux）

```bash
$ umask
022
```

---

## ファイル作成例

```bash
touch test.txt
ls -l
```

結果

```text
-rw-r--r--
```

計算すると

```
666
-022
----
644
```

となる。

ディレクトリの場合は

```
777
-022
----
755
```

となる。

---

## umaskはどこで決まる？

umaskはLinuxカーネルが決めているわけではなく、

- PAM
- `/etc/login.defs`
- `/etc/profile`
- `/etc/bashrc`
- `~/.bashrc`

などで設定される。

ディストリビューションによってデフォルト値は異なるが、

RHEL系（AlmaLinux・Rocky・CentOS）では `022` が一般的。

確認例

```bash
grep UMASK /etc/login.defs
```

# 所有者・所有グループ

Linuxではすべてのファイルに

- 所有者（Owner）
- 所有グループ（Group）

が設定されている。

確認

```bash
ls -l
```

例

```text
-rw-r--r-- 1 haruka haruka 0 Jul 14 test.txt
```

```
所有者 : haruka
グループ : haruka
```

---

# アクセス権

権限は

```
r
w
x
```

の3種類。

|権限|意味|
|---|---|
|r|読み取り|
|w|書き込み|
|x|実行|

表示例

```text
-rwxr-xr--
```

```
所有者 : rwx
グループ : r-x
その他 : r--
```

---

# 特殊権限

## SUID

実行したユーザーではなく、

**ファイル所有者の権限**で実行される。

例

```text
passwd
```

一般ユーザーでもパスワード変更できる理由。

---

## SGID

### ファイル

所有グループの権限で実行される。

### ディレクトリ

そのディレクトリ内で作成されたファイルは

親ディレクトリのグループを引き継ぐ。

---

## Sticky Bit

ディレクトリ内では

**作成者本人だけが削除できる。**

代表例

```text
/tmp
```

# ファイル管理

ファイルやディレクトリの管理では

- cp
- mv
- rm
- mkdir
- touch

などの基本コマンドを利用する。

---

# -R オプション

多くのLinuxコマンドでは

```bash
-R
```

は

```
Recursive（再帰的）
```

を意味する。

例

```bash
chmod -R
chown -R
cp -R
```

など。

---

## 再帰的（Recursive）とは

ある処理の中で、自分自身を繰り返し処理すること。

フォルダ構造でいうと、

```
親フォルダ
 ├── 子フォルダ
 │     ├── 孫フォルダ
 │     │      └── …
```

のように、一番奥まで同じ処理を繰り返すことを指す。

例えば

```bash
chmod -R 755 dir
```

は

- dir
- dir配下のすべてのディレクトリ
- dir配下のすべてのファイル

に対して権限変更を行う。

「鏡の中に鏡が映り続ける」ような自己参照・入れ子構造をイメージすると理解しやすい。

---

# 学び

- Linuxではファイルとディレクトリでデフォルト権限が異なる（666 / 777）
- umaskは「引き算」で権限を決める
- umaskの値はディストリビューションや設定ファイルによって決まる
- 所有者・所有グループ・アクセス権は `ls -l` で確認できる
- `-R` は Recursive（再帰的）の略で、サブディレクトリまで処理を適用するオプションとして頻繁に使われる
