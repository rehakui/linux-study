# TCP/IP Overview

## TCP/IP

A protocol stack used for communication on the Internet.

## OSI Model

A theoretical network model with seven layers.  
It was not widely adopted but is useful for learning networking.

## TCP/IP Model

A practical network model developed in the United States.  
It became the de facto standard with the growth of the Internet.

## Internet

A WAN (Wide Area Network) built on the TCP/IP protocol suite.

## TCP/IP Layers

The TCP/IP model has four layers:

- Application
    
- Transport
    
- Internet
    
- Link
    

## RFC (Request for Comments)

Documents that define Internet standards and technical specifications.

## TCP (Transmission Control Protocol)

Provides reliable and connection-oriented communication.  
Data is sent in units called **segments**.

## Control Bits

Flags used to control TCP communication.  
They are mainly used for connection management and congestion control.

## UDP (User Datagram Protocol)

Provides fast and connectionless communication.  
Data is sent in units called **datagrams**.

## IP (Internet Protocol)

Delivers packets from end to end using IP addresses.  
It provides basic delivery, while other protocols add extra features if needed.

## IP Address

A unique address used to identify a device on a network.

## IPv4

Uses 32-bit addresses (2³² ≈ 4.3 billion addresses).  
The limited address space led to IPv4 address exhaustion.

## IPv6

The successor to IPv4 with a much larger address space.

## Network Address & Host Address

An IP address consists of a network part and a host part.

## Public IP Address

A globally unique address used on the Internet.

## Private IP Address

An address used inside a local network.

## NAT (Network Address Translation)

Converts private IP addresses into public IP addresses.

## PAT (Port Address Translation)

Allows multiple devices to share one public IP address by using different port numbers.

---

## Key Takeaway

- TCP/IP is the practical protocol suite that powers the Internet.
    
- TCP provides reliable communication, while UDP focuses on speed.
    
- IP delivers data using IP addresses, and NAT/PAT help solve the IPv4 address shortage.
