# 問われた内容・回答

## 1. CPUの情報を確認する方法

### 問題

CPUの詳細情報を確認するにはどのコマンドを使用するか。

### 回答

```bash
cat /proc/cpuinfo
```

または

```bash
lscpu
```

### 学んだこと

- `/proc/cpuinfo` はCPUの詳細情報を表示する仮想ファイル。
- `lscpu` はCPU情報を見やすく一覧表示してくれる。

実際に以下の内容を確認した。

- CPUモデル
- CPUアーキテクチャ
- CPUコア数
- キャッシュサイズ
- 仮想化機能

---

## 2. カーネルモジュールを確認する方法

### 問題

現在読み込まれているカーネルモジュールを確認するには？

### 回答

```bash
lsmod
```

### 学んだこと

- モジュール名
- サイズ
- 使用回数

を一覧で確認できる。

---

## 3. ディスク・パーティションを確認する方法

### 問題

ディスクやパーティション構成を確認するには？

### 回答

```bash
sudo fdisk -l
```

### 学んだこと

以下を確認できた。

- ディスク容量
- パーティション番号
- パーティションサイズ
- パーティションタイプ

例

```
/dev/sda1
/dev/sda2
/dev/sda3
```

---

## 4. パーティションとは

### 問題

パーティションとは何か。

### 回答

1台のディスクを複数の領域に分割したもの。

用途ごとに

- OS
- データ
- Swap

などを分けて管理できる。

---

## 5. ファイルシステムとは

### 問題

ファイルシステムとは何か。

### 回答

ディスク上でファイルやディレクトリを管理する仕組み。

代表例

- ext4
- xfs
- FAT32
- NTFS

---

## 6. マウントとは

### 問題

マウントとは何か。

### 回答

ストレージをLinuxのディレクトリツリーへ接続して利用できるようにすること。

アンマウントすると利用できなくなる。

---

## 7. スワップ領域とは

### 問題

スワップ領域の役割は？

### 回答

メモリ不足時に補助メモリとして使用するディスク領域。

# 気づき

- `/proc`は実際のファイルではなく、カーネル情報を表示する仮想ファイルシステムであることを理解した。
- `lscpu`の方がCPU情報を整理して確認できるため使いやすかった。
- `fdisk -l`でディスク構成やパーティションを実際に確認でき、Linuxがどのようにストレージを管理しているか理解できた。
- パーティション・ファイルシステム・マウントはそれぞれ役割が異なることを整理できた。
- Linuxではストレージを利用するために「パーティション作成 → ファイルシステム作成 → マウント」という流れになることを理解した。
