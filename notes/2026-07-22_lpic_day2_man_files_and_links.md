# 学習内容

## 1. manコマンド

### 問題

Linuxコマンドのマニュアルを確認するには？

### 回答

```bash
man コマンド名
```

例

```bash
man passwd
man printf
```

### 学んだこと

- Linuxにはオンラインマニュアル(manページ)が用意されている。
- コマンドの概要だけでなく、オプションや使用例も確認できる。
- 分からないコマンドはまず `man` を見る習慣を付けたい。

---

## 2. whatisコマンド

### 問題

コマンドの概要だけを簡単に確認したい場合は？

### 回答

```bash
whatis passwd
whatis printf
```

出力例

```
passwd (1) - change user password
printf (1) - format and print data
```

### 学んだこと

- `man` よりも簡潔に説明を確認できる。
- AlmaLinuxでは `sudo` を付けないとデータベースが利用できない環境もあることを確認した。

---

## 3. ヘルプ表示

### 問題

コマンドのオプション一覧を表示するには？

### 回答

```bash
passwd -h
```

### 学んだこと

- 多くのLinuxコマンドは `-h` や `--help` を利用できる。
- `man` より素早くオプションを確認できる。

---

## 4. lsコマンド

### 問題

詳細表示をするには？

### 回答

```bash
ls -l
```

### 学んだこと

誤って

```bash
ls --long
```

を実行したところ

```
unrecognized option '--long'
```

となった。

Linuxでは

```bash
ls -l
```

を使用する。

---

## 5. inode番号

### 問題

inode番号を確認するには？

### 回答

```bash
ls -i
```

### 学んだこと

inode番号はファイルの実体を管理する番号であり、ハードリンクは同じinode番号を持つ。

---

## 6. 実行ファイルの場所

### 問題

コマンド本体の場所を確認するには？

### 回答

```bash
ls /bin
```

例

```bash
ls /bin/ls
```

### 学んだこと

Linuxコマンドは `/bin` や `/usr/bin` に配置されている。

---

## 7. ファイルの内容を表示

### 問題

ファイル内容を表示するには？

### 回答

```bash
cat ファイル名
```

例

```bash
cat test.txt
cat /etc/services
```

### 学んだこと

- `cat` はテキストファイルの確認に使用する。
- ディレクトリを指定すると

```
Is a directory
```

となる。

---

## 8. リダイレクト

### 問題

コマンドの出力をファイルへ保存するには？

### 回答

```bash
date > today
```

### 学んだこと

- `>` は標準出力をファイルへ保存する。
- 既存ファイルは上書きされる。

---

## 9. ハードリンク

### 問題

ハードリンクを作成するには？

### 回答

```bash
ln today today.hardl
```

### 学んだこと

- 同じinode番号を共有する。
- 元ファイルと同じ実体を指す。

---

## 10. シンボリックリンク

### 問題

シンボリックリンクを作成するには？

### 回答

```bash
ln -s today today.syml
```

### 学んだこと

- ショートカットのような存在。
- `ls -l` では

```
today.syml -> today
```

と表示される。

# 理解したポイント

## man

- Linux公式マニュアル

## whatis

- コマンド概要のみ表示

## cat

- ファイル内容表示

## リダイレクト

```
>
```

でファイルへ保存

## ハードリンク

- inode共有
- 元ファイルと同じ実体

## シンボリックリンク

- パスへの参照
- Windowsのショートカットに近い
