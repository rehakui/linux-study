# 21. ジョブの管理

## ジョブとは

Linuxでシェルが管理する実行中のプロセスのこと。

ジョブには2種類ある。

- フォアグラウンドジョブ
- バックグラウンドジョブ

---

## フォアグラウンドジョブ

### 特徴

- ターミナルを占有する
- キーボード入力を受け付ける
- 終了するまで他のコマンドを実行できない

### 例

```bash
vim file.txt
```

---

## バックグラウンドジョブ

### 特徴

- ターミナルを占有しない
- 他のコマンドを実行できる
- 「&」を付けて実行する

### 例

```bash
sleep 100 &
```

---

## ジョブ制御コマンド

### ジョブ一覧

```bash
jobs
```

### ジョブを一時停止

```text
Ctrl + Z
```

### バックグラウンドで再開

```bash
bg
```

### フォアグラウンドへ戻す

```bash
fg
```

### ジョブ終了

```bash
kill %1
```

---

## 試験ポイント

| コマンド | 役割 |
|----------|------|
| jobs | ジョブ一覧表示 |
| bg | バックグラウンド実行 |
| fg | フォアグラウンドへ戻す |
| Ctrl + Z | ジョブを一時停止 |

# X Window System

## 一言でいうと

LinuxでGUI（画面表示）を実現する仕組み。

---

## X Window Systemの役割

- ウィンドウ表示
- ディスプレイ管理
- キーボード管理
- マウス管理
- 入出力装置の管理

---

## Xサーバ（超重要）

### 役割

**ディスプレイ・キーボード・マウスなどの入出力装置を管理する。**

つまり、

> 実際に画面へ表示し、入力を受け付ける側

### 管理するもの

- ディスプレイ
- キーボード
- マウス

---

## Xクライアント

GUIアプリケーションのこと。

### 例

- Chrome
- Firefox
- Terminal
- LibreOffice

### 役割

Xサーバへ

「画面に表示してください」

と依頼する。

---

## イメージ

```text
Chrome
Firefox
Terminal
    │
    ▼
Xクライアント
    │
    ▼
Xサーバ
    │
 ┌──┼──┐
 │  │  │
画面 キーボード マウス
```

---

## Windowsとの違い

### Windows

```text
Excel
  │
Windows
  │
画面
```

GUI機能がOSに組み込まれている。

### Linux

```text
Chrome
   │
X Window System
   │
画面
```

GUI機能がOSから独立している。

---

## Waylandとの比較

| X11 | Wayland |
|------|----------|
| 古い仕組み | 新しい仕組み |
| 機能が多い | シンプル |
| 互換性が高い | 高速 |
| 現在も利用される | 現在の主流 |

---

## 試験ポイント

### Xサーバ

- 入出力装置を管理する
- ディスプレイ・キーボード・マウスを管理

### Xクライアント

- GUIアプリケーション
- Xサーバへ描画を依頼する

### 覚え方

> **サーバ = 画面側**
>
> **クライアント = アプリ側**

# 23. X Window Systemの利用

## DISPLAY環境変数

接続するXサーバを指定する環境変数。

### 確認

```bash
echo $DISPLAY
```

### 例

```text
:0
```

または

```text
localhost:10.0
```

---

## xauth

Xサーバ接続用の認証情報（Cookie）を管理する。

### 一覧表示

```bash
xauth list
```

---

## xhost

Xサーバへの接続許可を管理する。

### 例

```bash
xhost +
```

※ セキュリティ上は推奨されない。

---

## SSHでX転送

### 接続

```bash
ssh -X user@host
```

または

```bash
ssh -Y user@host
```

リモートサーバ上のGUIアプリをローカル画面に表示できる。

# 小テスト3で出題されたポイント

- ジョブとは、シェルが管理する実行中のプロセス
- フォアグラウンドジョブはターミナルを占有する
- バックグラウンドジョブはターミナルを占有しない
- X Window SystemはLinuxのGUIを提供する仕組み
- **Xサーバは入出力装置（ディスプレイ・キーボード・マウス）を管理する**
- XクライアントはGUIアプリケーション
- DISPLAY環境変数は接続するXサーバを指定する
- `echo $DISPLAY`で確認できる
- `xauth list`は認証情報（Cookie）の一覧表示

# 覚え方

```text
GUIアプリ
（Chrome・Firefox）
        │
        ▼
   Xクライアント
        │
        ▼
      Xサーバ
        │
 ┌─────┼─────┐
 │     │     │
画面 キーボード マウス
```

**ポイント**

- アプリが「表示したい」と依頼するのが **Xクライアント**
- 実際に画面表示・入力管理を行うのが **Xサーバ**
- **「サーバ＝画面側」「クライアント＝アプリ側」** と覚えると試験で迷わない。
