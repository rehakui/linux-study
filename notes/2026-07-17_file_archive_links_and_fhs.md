## 問われた内容・学んだこと

### アーカイブの作成と展開

- アーカイブは複数のファイルを1つにまとめる。
- 圧縮とは役割が異なり、必要に応じて組み合わせて利用する。

### ハードリンクとシンボリックリンク

#### ハードリンク

```bash
ln 元ファイル リンク名
```

- 元ファイルと同じinode番号を持つ。
- ファイル名が増えるイメージ。
- 元ファイルを削除しても、ハードリンクが残っていればデータは参照できる。

#### シンボリックリンク

```bash
ln -s 元ファイル リンク名
```

- 元ファイルへのショートカット。
- inode番号は異なる。
- 元ファイルを削除するとリンク切れになる。

### リンクの見分け方

```bash
ls -l
```

- シンボリックリンクは

```text
リンク名 -> 元ファイル
```

の形式で表示される。

```bash
ls -i
```

- inode番号を確認できる。
- ハードリンクは同じinode番号になる。

```bash
ls -F
```

- シンボリックリンクには末尾に`@`が付く。

### ファイル検索

```bash
find 検索開始ディレクトリ -name "ファイル名"
```

例

```bash
find /home/haruka/test -name "t*"
```

- `find`は指定したディレクトリ以下を検索する。

#### メタキャラクタ

- `*`：任意の文字列
- `?`：任意の1文字

### FHS（Filesystem Hierarchy Standard）

Linuxで使用されるディレクトリ構成の標準。

| ディレクトリ | 役割 |
|--------------|------|
| `/bin` | 一般ユーザー向けコマンド |
| `/sbin` | 管理者向けコマンド |
| `/etc` | 設定ファイル |
| `/dev` | デバイスファイル |
| `/lib` | ライブラリ |
| `/media` | USBなどの外部メディア |
| `/proc` | カーネル・プロセス情報（仮想ファイル） |
| `/home` | 一般ユーザーのホームディレクトリ |
| `/root` | rootユーザーのホームディレクトリ |
| `/boot` | 起動に必要なファイル |
| `/tmp` | 一時ファイル |
| `/var` | ログや可変データ |
| `/usr` | コマンド・ライブラリ・マニュアルなど |

## 実践

- ハードリンクとシンボリックリンクを作成した。
- `ls -l`、`ls -i`、`ls -F`を使って両者の違いを確認した。
- `find`コマンドとワイルドカードを使ってファイル検索を実施した。
- FHSの主要ディレクトリと役割を学習した。

## 気づき

- ハードリンクは「同じデータを指す別名」、シンボリックリンクは「ショートカット」というイメージで理解すると分かりやすかった。
- `ls -i`でinode番号を見るとハードリンクとシンボリックリンクの違いが明確に分かる。
- FHSは暗記するだけでなく、「どんなファイルが置かれる場所なのか」を理解して覚えることが重要だと感じた。
