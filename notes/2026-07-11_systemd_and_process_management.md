# systemdとプロセス管理

2026-07-11

## 学習内容

- Udemy Section 2.19-2.20

### systemd

デフォルトターゲットの確認

```bash
systemctl get-default
```

`default.target`がどのターゲットを指しているか確認

```bash
ls -l default.target
```

### プロセス管理

現在の端末で実行中のプロセスを表示

```bash
ps a
```

システム全体のプロセスを表示

```bash
ps ax
```

コマンドの実行時間を計測

```bash
time
```

CPU・メモリ使用率や実行中のプロセスをリアルタイムで監視

```bash
top
```

## 実践

- `systemctl get-default`で現在のデフォルトターゲットが`multi-user.target`であることを確認した。
- `default.target`が`multi-user.target`へのシンボリックリンクであることを確認した。
- `ps a`と`ps ax`を実行し、表示されるプロセスの違いを確認した。
- `time`コマンドでコマンドの実行時間を確認した。
- `top`を実行し、CPU使用率・メモリ使用量・実行中のプロセスをリアルタイムで確認した。

## 学び・気づき

- `systemd`では`default.target`がシンボリックリンクとなっており、現在の起動ターゲットを確認できることを理解した。
- `ps`はオプションによって表示範囲が異なり、`ps a`は現在の端末に関連するプロセス、`ps ax`はシステム全体のプロセスを表示する。
- `top`はシステムの状態をリアルタイムで監視でき、CPU・メモリ使用率や負荷状況の確認に役立つ。
- コマンドを暗記するだけでなく、実際に実行して結果を確認することで理解が深まると感じた。
