## 問われた内容・学んだこと

### シェルの基本

- シェルは、ユーザーとOSをつなぐインターフェース。
- 入力したコマンドを解釈し、OSへ実行を依頼する。

---

### シェル変数

変数の代入

```bash
shark=12
```

変数の参照

```bash
echo $shark
```

変数の削除

```bash
unset shark
```

---

### 環境変数

環境変数は子プロセスにも引き継がれる。

設定

```bash
export tiger=3
```

確認

```bash
echo $tiger
```

主な環境変数

|環境変数|内容|
|--------|----|
|USER|ユーザー名|
|HOME|ホームディレクトリ|
|PATH|コマンド検索パス|
|LANG|言語設定|
|EDITOR|標準エディタ|

確認例

```bash
echo $USER
echo $LANG
echo $EDITOR
```

---

### コマンド実行

現在のユーザー

```bash
who am i
```

実行履歴

```bash
history
```

コマンドの場所

```bash
which vi
which ls
```

---

### マニュアルの参照

コマンドの使い方を確認する。

```bash
man passwd
man cal
```

---

### テキストストリーム

Linuxではコマンドの入出力をテキストとして扱う。

例

```bash
ls | wc
```

出力

- 行数
- 単語数
- バイト数

---

### パイプ

あるコマンドの出力を次のコマンドへ渡す。

```bash
ls test | wc
```

---

### リダイレクト

出力をファイルへ保存する。

```bash
ls test > test/ls.txt
```

内容確認

```bash
cat test/ls.txt
```

---

### grepによる検索

文字列検索

```bash
grep Hello test/test.txt
```

大文字・小文字を区別しない

```bash
grep hello test/test.txt
```

部分一致

```bash
grep ello test/test.txt
```

一致件数のみ表示

```bash
grep -c Hello test/test.txt
```

---

### 正規表現

grepでは正規表現を利用できる。

例

```bash
grep "^H" test.txt
grep "d$" test.txt
```

---

### エディタ（vi）

インストール確認

```bash
which vi
```

vimが存在しない場合はインストール

```bash
sudo dnf install vim
```

ファイル編集

```bash
vi sample.py
```

Pythonファイルを作成し、簡単なプログラムを保存・編集した。

---

## 実践

- シェル変数と環境変数を作成・削除した。
- `export`で環境変数を設定した。
- `history`や`which`でコマンド情報を確認した。
- `man`でコマンドマニュアルを参照した。
- `|`（パイプ）を使ってコマンドを連結した。
- `>`（リダイレクト）で出力をファイルへ保存した。
- `grep`で文字列検索を行った。
- `vi`を利用してPythonファイルを編集した。
- vimが未インストールだったため、`dnf`でインストールして利用できることを確認した。

## 気づき

- シェル変数と環境変数は似ているが、環境変数は子プロセスへ引き継がれる点が大きな違いだった。
- `echo $変数名`で変数の確認がすぐできる。
- `which`はコマンドの実体を調べる際に便利だった。
- パイプとリダイレクトを組み合わせることで、コマンド同士を柔軟につなげられることを理解した。
- `grep`はLinuxで最も利用頻度の高い検索コマンドの一つであり、正規表現と組み合わせることで強力な検索ができる。
- `vi`は最初は操作が難しいが、Linuxでは必須のエディタなので少しずつ慣れていきたい。
