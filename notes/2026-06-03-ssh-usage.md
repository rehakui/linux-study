# 2026-06-03 SSHの利用

## 学習内容

SSHを利用する際に関係するファイルと基本コマンドについて学習した。

---

## クライアント側で関係するSSHファイル

これらのファイルはホームディレクトリ直下の `.ssh` ディレクトリに保存される。

### ssh_config

* クライアント側のSSH設定ファイル

### id_rsa

* SSHの秘密鍵
* 外部に漏らしてはいけない

### id_rsa.pub

* SSHの公開鍵
* サーバーへ登録する

### known_hosts

* 接続先サーバーの情報を保存するファイル
* ホスト認証に使用される公開鍵が保存される

---

## サーバー側で関係するSSHファイル

主に `/etc/ssh` ディレクトリに保存される。

### sshd_config

* SSHサーバーの設定ファイル

### ssh_host_rsa_key

* サーバーの秘密鍵

### ssh_host_rsa_key.pub

* サーバーの公開鍵

### authorized_keys

* 接続を許可するクライアントの公開鍵リスト
* クライアントが登録した公開鍵を保存する

※ `authorized_keys` のみユーザーのホームディレクトリ直下の `.ssh` ディレクトリに保存される。

---

## SSHで利用する基本コマンド

### 1. 公開鍵と秘密鍵のペアを作成

```bash
ssh-keygen
```

* SSHの公開鍵と秘密鍵を作成する

---

### 2. 公開鍵をリモートホストへ登録

```bash
ssh-copy-id ユーザー名@ホスト名
```

例

```bash
ssh-copy-id user@example.com
```

* 公開鍵をサーバーへ登録する
* 登録時はパスワード認証など他の認証方式が有効である必要がある

---

### 3. リモートホストへログイン

```bash
ssh ユーザー名@ホスト名
```

例

```bash
ssh user@example.com
```

または

```bash
ssh user@192.168.1.10
```

* SSHでリモートホストへ接続する

---

### 4. リモートホストからログアウト

以下のいずれかを実行する。

```bash
logout
```

```bash
exit
```

または

```text
Ctrl + D
```

---

## 学んだこと

* SSHにはクライアント側とサーバー側で管理されるファイルがある
* クライアント側の設定や鍵は `.ssh` ディレクトリに保存される
* 公開鍵認証では公開鍵をサーバーの `authorized_keys` に登録する
* `ssh-keygen` で鍵ペアを作成できる
* `ssh-copy-id` で公開鍵をサーバーへ登録できる
* `ssh` コマンドでリモートログインできる
* `exit` や `logout`、`Ctrl + D` で接続を終了できる
