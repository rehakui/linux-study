## シェルの基本

### OS

OSの最も重要な役割は、アプリケーションとハードウェアを管理すること。

### カーネル

OSの中核部分。

ハードウェアを管理しているが、ユーザが直接操作することはできない。

### シェル

ユーザとカーネルをつなぐ通訳係。

ユーザが入力したコマンドを解釈し、カーネルへ伝える。

---

## コマンド実行の流れ

1. プロンプト（`$` または `#`）を表示
2. コマンドを入力
3. コマンドを検索して実行
4. 実行結果を表示
5. プロセス終了
6. 次の入力を待つ

この流れを繰り返す。

---

## シェルの種類

| シェル  | 概要             |
| ---- | -------------- |
| sh   | Unix標準シェル      |
| bash | shを拡張した標準的なシェル |
| csh  | C言語風の文法        |
| tcsh | cshの拡張版        |
| ksh  | shの拡張版         |
| zsh  | 高機能なシェル        |

## 変数

変数には値を保存できる。

```bash
変数名=値
```

参照するときは `$変数名` を使用する。

---

## シェル変数と環境変数

### シェル変数

変数を定義したシェル内だけで有効。

### 環境変数

システムやプログラムが利用する設定値。

代表例

* PATH
* LANG
* HISTFILESIZE

---

## パス

ファイルやディレクトリの場所を表す。

### 絶対パス

ルートディレクトリ（`/`）から記述する。

例

```text
/home/user/file.txt
```

### 相対パス

現在のディレクトリを基準に記述する。

例

```text
documents/file.txt
```

---

## 内部コマンドと外部コマンド

### 内部コマンド

シェルに組み込まれているコマンド。

例

* cd
* history

### 外部コマンド

独立したプログラムとして存在するコマンド。

例

* ls
* cp
* mv

---

## コマンド履歴

### history

実行したコマンド履歴を表示する。

履歴番号を利用して再実行できる。

```bash
history
```

## パイプとリダイレクト

### 標準入力（stdin）

プログラムへの入力。

通常はキーボード。

### 標準出力（stdout）

正常な実行結果の出力先。

通常は画面。

### 標準エラー出力（stderr）

エラーメッセージの出力先。

通常は画面。

### ストリーム

データの流れ。

### リダイレクト

入力元や出力先を変更する仕組み。

例

```bash
ls > result.txt
```

### パイプ

コマンドの出力を別のコマンドの入力として渡す仕組み。

例

```bash
ls | grep txt
```
