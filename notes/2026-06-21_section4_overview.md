## マニュアルの参照

### manコマンド

コマンドの説明書（マニュアル）を表示する。

```bash
man ls
```

---

## テキスト処理

### テキストフィルタコマンド

入力されたテキストを加工して出力するコマンド。

例

* grep
* sort
* cut
* wc

---

## パイプとリダイレクト

### 標準入力（stdin）

プログラムへの入力。

通常はキーボード。

### 標準出力（stdout）

正常な実行結果の出力先。

通常は画面。

### 標準エラー出力（stderr）

エラーメッセージの出力先。

通常は画面。

### ストリーム

データの流れ。

### リダイレクト

入力元や出力先を変更する仕組み。

例

```bash
ls > result.txt
```

### パイプ

コマンドの出力を別のコマンドの入力として渡す仕組み。

例

```bash
ls | grep txt
```

## テキストファイルの編集

### viエディタ

Linux標準のテキストエディタ。

### vim

viを高機能化したエディタ。

### Emacs

高機能な別系統のエディタ。

### 入力モード

文字を入力するモード。

### コマンドモード

保存・終了・検索などを行うモード。

---

## メモ

今回はテキスト処理やエディタ操作の概要を理解した。

細かなコマンドやオプションを覚える段階ではなく、

* manで調べる
* パイプでコマンドをつなぐ
* リダイレクトで出力先を変える
* 正規表現で検索する
* vi/vimでファイルを編集する

といった全体像を把握することを優先した。
